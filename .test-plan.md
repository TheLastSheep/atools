## Completed Security and Reliability Boundaries

- Main and hosted plugin documents use `sandbox="allow-scripts allow-popups"`; neither receives `allow-same-origin`.
- Plugin invoke handling is deny-by-default, bound to the active source and generation, rebuilds command arguments, and forces the active plugin ID.
- Plugin install/update/import uses a serialized same-parent staged directory swap with byte rollback when database persistence fails; uninstall quarantines first and restores on database failure.
- Source/destination symlinks are rejected. Market ZIPs reject encrypted/symlink/unsafe/duplicate entries and enforce archive, entry-count, path-depth, per-entry, and total-uncompressed quotas.
- Marketplace installation requires a valid Ed25519 signature and a public key matching a local settings/environment pin. Catalog-supplied keys cannot self-anchor, and local pins are excluded from WebDAV backup/restore.
- Plugin metadata/features update in one database transaction without `REPLACE` on `plugins` or `plugin_data`; existing attachment/data retention is regression-tested.
